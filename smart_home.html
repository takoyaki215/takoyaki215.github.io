Design Conceptualization

Knowing the importance of aesthetics and practicability one asks, “how does this app fit into the lives of the users?” However, the UI of the first version of a typical air conditioner app is created by engineers unfamiliar with user interface. Thus, ideal user experiences, user concerns and behavior must be addressed. As a result, I set both short-term and long-term design goals.

▾ Before/ After

Air Conditioner

For the short-term goal, to improve the aesthetics is the most important issue in regard to the air conditioner app. Therefore, all of the positions and functions in the app are the same as before, I only modified the color and icons of the UI, and then I added a tutorial during usage making the app more user-friendly.

▾ AC Mockup

Dehumidifier

Practicability is a major task for an app, thus I set up concordance among three appliances: the air conditioner, dehumidifier, and air purifier, as my long-term goal. Additionally, I added an “admin” to manage other household users, so they can switch to any of the appliances.

appliance management:
Through this function, users can switch to any of the appliances, and the admin interface page shows the ambient household temperature, humidity, and air quality as well.
user management:
This function is accessible for only the admin user, to decide who else can access these appliances.
